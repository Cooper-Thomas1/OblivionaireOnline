Handles user login authentication and session setup.

Implements the business logic for user login:

  • Looks up the account by user ID.
  • Checks for account ban or expiration.
  • Checks for excessive failed login attempts.
  • Validates the supplied password.
  • Records login success or failure.
  • Populates the session struct on success.
  • Sends messages to the client and logs events.
Parameters
useridNull-terminated user ID string (must not be NULL).
passwordNull-terminated password string (must not be NULL).
client_ipIPv4 address of the client.
login_timeTimestamp of the login attempt.
client_output_fdWritable file descriptor for client messages.
sessionPointer to session struct to populate on success (must not be NULL).
Returns
LOGIN_SUCCESS on success, or appropriate LOGIN_FAIL_* code on failure.
Precondition
userid, password, and session must be non-NULL and valid null-terminated strings.
client_output_fd must be a valid, writable file descriptor.
